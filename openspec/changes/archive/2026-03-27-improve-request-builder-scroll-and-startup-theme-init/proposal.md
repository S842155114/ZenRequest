## Why

随着请求构建内容变多，参数编辑区在视觉和交互上会挤压到下方响应区，导致请求编排体验不稳定。与此同时，应用启动时首屏主题先按系统值绘制、再被运行时设置覆盖，用户会看到明显的浅深色闪烁。

## What Changes

- 调整请求工作台的上半区布局，让请求标签栏和 URL 操作栏保持稳定，参数构建区作为独立纵向滚动区域承载长内容。
- 约束请求构建区与响应区之间的布局边界，避免参数、请求头、鉴权、请求体等内容增长时出现被响应区遮挡的感知。
- 在启动文档首帧就根据已持久化的主题模式预设主题，并让启动占位与应用接管后的主题保持一致。
- 调整启动流程中启动屏的移除时机，确保运行时设置与主题已经生效后再退出启动态。

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `workbench-ui`: 调整请求构建区域的滚动与布局要求，确保请求编排区在长内容场景下保持独立可滚动且不与响应区发生视觉冲突。
- `startup-experience`: 调整启动阶段的主题连续性要求，确保首屏、启动占位和运行时主题在 bootstrap 完成前后保持一致。

## Impact

- 前端布局组件与滚动容器，包括请求工作台、请求参数区域及相关样式。
- 应用启动入口与启动屏切换逻辑，包括 `index.html`、前端 bootstrap 流程与主题初始化逻辑。
- 与请求工作台滚动行为、启动主题初始化相关的前端测试与验证流程。
