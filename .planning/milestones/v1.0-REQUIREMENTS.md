# Requirements Archive: ZenRequest v1.0

**Archived:** 2026-04-06
**Milestone:** v1.0
**Outcome:** ✅ All v1 requirements satisfied and audit-closed

## v1 Requirements

### Core Request Flow

- [x] **CORE-01**: 开发者可以在单个工作区中快速创建、编辑并重复发送 HTTP 请求
- [x] **CORE-02**: 开发者可以查看响应状态码、耗时、响应头、原始响应和格式化响应内容
- [x] **CORE-03**: 开发者在重启应用后可以恢复最近工作区、标签页和当前调试上下文
- [x] **CORE-04**: 开发者在处理大响应体、文件上传和频繁重发时不会遇到明显卡顿、崩溃或历史损坏

### Workspace And Assets

- [x] **WS-01**: 开发者可以保存请求到 collection，并在 collection 结构中管理常用请求资产
- [x] **WS-02**: 开发者可以查看、筛选并重新发送历史请求记录
- [x] **WS-03**: 开发者可以导出与导入本地工作区或关键请求资产，用于备份和迁移
- [x] **WS-04**: 开发者可以从 cURL 导入请求草稿并继续编辑

### Variables And Auth

- [x] **VAR-01**: 开发者可以定义并使用环境变量与模板值来复用 base URL、token 和其他参数
- [x] **VAR-02**: 开发者可以明确理解变量解析优先级，并获得稳定一致的解析结果
- [x] **AUTH-01**: 开发者可以对请求配置 No Auth、Basic Auth、Bearer Token 与 API Key
- [x] **AUTH-02**: 开发者在导出、分享或迁移工作区时不会意外泄露敏感 secret 值

### Testing And Reliability

- [x] **TEST-01**: 开发者可以为请求添加基础断言测试，并在请求执行后查看断言结果
- [x] **TEST-02**: 开发者在本地数据库损坏、数据恢复失败或历史记录异常时可以得到明确诊断与可执行恢复提示
- [x] **TEST-03**: 开发者在请求执行、导入或持久化失败时可以看到结构化且可定位的错误信息

### MCP Workbench

- [x] **MCP-01**: 开发者可以把 MCP Server 作为独立目标进行 `initialize`、`tools.list` 与 `tools.call` 调试
- [x] **MCP-02**: 开发者可以查看 MCP 调用的结构化结果、原始协议包、历史摘要与回放记录
- [x] **MCP-03**: 开发者可以在 schema 驱动表单与 raw JSON 之间切换，完成工具参数输入
- [x] **MCP-04**: 开发者在 MCP 调试失败时可以看到 transport、session 或 tool-call 层面的可定位错误上下文
