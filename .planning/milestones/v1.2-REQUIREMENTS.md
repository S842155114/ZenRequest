# Requirements: ZenRequest

**Defined:** 2026-04-10
**Core Value:** 让开发者以本地优先、快速、可控的方式完成高频 API 调试与工作流操作，而不被臃肿云平台和账号绑定打断。

## v1.2 Requirements

### In-App Guidance

- [x] **GUIDE-01**: 开发者可以在设置中找到稳定可见的帮助入口
- [x] **GUIDE-02**: 开发者在 MCP `stdio` 首次使用时可以看到空状态引导，明确如何完成第一次成功连接
- [x] **GUIDE-03**: 开发者可以在产品内看到 `stdio` 关键字段的填写说明与常见错误提示

### Documentation System

- [x] **DOCS-01**: 仓库 `README` 提供快速上手入口，并链接到完整文档
- [x] **DOCS-02**: `docs/` 下提供中文教程型产品手册，覆盖主要功能与核心使用路径
- [x] **DOCS-03**: `docs/` 下提供英文教程型产品手册，与中文内容保持结构对应
- [x] **DOCS-04**: 关键功能文档包含截图，首轮以中文截图为主

### Information Architecture

- [x] **IA-01**: 产品内帮助入口、README、文档手册之间形成清晰跳转关系
- [x] **IA-02**: 文档内容覆盖 HTTP、MCP、导入、历史/回放、`stdio` 等当前主线能力

## v2 Requirements

### MCP Expansion

- **MCPS-01**: 开发者可以调试 MCP `sampling` 能力，并检查其输入输出与安全边界
- **MCPS-02**: 开发者可以管理多个本地或远端 MCP server，并在它们之间切换、隔离历史和会话

### AI / Agent Workflows

- **AGNT-01**: 开发者可以基于文档、说明或历史请求生成可编辑请求与测试草稿
- **AGNT-02**: 开发者可以回放和检查 agent tool-call 轨迹，包括输入、schema 快照和输出差异
- **AGNT-03**: 开发者可以运行轻量的混合 HTTP + MCP 场景流，并在本地查看执行轨迹

## Out of Scope

| Feature | Reason |
|---------|--------|
| 仅为 `stdio` 做一次性临时提示而不建设完整文档体系 | 本 milestone 目标是建立长期可维护的使用说明体系 |
| 首轮中英分别重拍整套截图 | 当前优先级是先把中英文档结构和核心内容落地，英文可先复用中文截图 |
| 在本 milestone 内新增 MCP `sampling` 或多 server 功能 | 本轮重点是可用性与文档体系，不是新增协议能力 |
