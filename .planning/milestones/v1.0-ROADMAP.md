# Milestone v1.0: ZenRequest MVP Foundation

**Status:** ✅ SHIPPED 2026-04-06
**Phases:** 1-7
**Total Plans:** 8

## Overview

v1.0 将 ZenRequest 从一个已有雏形的 brownfield 项目推进为可验证、可归档的本地优先桌面 API 工作台：HTTP 主链路稳定可用，工作区与资产迁移可依赖，变量与鉴权边界清晰，断言与恢复路径具备可信反馈，MCP over HTTP 工作台完成基础调试、回放和诊断闭环，并最终通过 Phase 6 / Phase 7 的 milestone re-audit 收口实现 archive-clean 状态。

## Milestone Summary

**Key Accomplishments:**
- 打通并稳定了本地优先 HTTP 请求编辑、发送、响应查看与会话恢复主链路。
- 收敛了 collection / history / import-export / cURL import 等核心资产能力。
- 完成环境变量、鉴权与 secret-safe export 的可依赖边界。
- 建立断言反馈、结构化错误和恢复指导的可信工作流。
- 交付 MCP over HTTP 的 initialize / tools.list / tools.call 基础工作台，并补齐 history / replay / diagnostics。
- 通过 Phase 6 / 7 关闭 archive-proof 与 milestone audit gaps，使 v1.0 达到 archive-ready。

**Key Decisions:**
- 以本地优先、离线优先、隐私优先和轻量快速作为 v1 的核心价值。
- 把 ZenRequest 定位为 AI / Agent 时代的本地 API 工作台，而不是功能追赶式“开源版 Postman”。
- 先打透 HTTP 主链路，再分阶段加固资产、变量、可靠性与 MCP 能力。
- 用 Phase 6 / 7 作为审计闭环 phase，而不是把 milestone gap 混入后续新特性里。

**Issues Resolved:**
- 工作区导入/恢复/历史异常的可见闭环与指引不足。
- MCP discovery continuity、schema lifecycle 与 taxonomy drift 的 milestone 审计问题。
- Phase 3 / 4 / 5 缺失 archive-proof 文档导致的归档阻塞。

**Issues Deferred:**
- MCP over stdio 与更广协议面。
- 更丰富的导入格式与 AI / Agent workflow 能力。
- 更完整的 MCP resources / prompts / roots / sampling 调试面。

**Technical Debt Incurred:**
- 无阻塞 v1.0 归档的新增技术债；未来扩展点主要来自刻意 deferred scope，而不是已知闭环缺口。
