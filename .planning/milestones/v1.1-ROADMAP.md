# Milestone v1.1: MCP Expansion

**Status:** ✅ SHIPPED 2026-04-10
**Phases:** 08-11
**Total Plans:** 4

## Overview

v1.1 将 ZenRequest 从基础 MCP tools 调试工作台扩展为更完整的单 server MCP 协议工作台，覆盖 `resources`、`prompts`、`roots` 与 `stdio` 传输，并继续保持统一的检查、历史、回放与诊断体验。

## Phases

### Phase 08: MCP Resources Workbench

**Goal**: 把 resources 查询与读取接入现有 MCP 工作台主链路
**Depends on**: Phase 07
**Plans**: 1 plan

Plans:

- [x] 08-01: Add resources discovery and read flow to MCP workbench

**Details:**
- 支持 `resources.list` 与 `resources.read`
- 保持 generic protocol/result/history/replay pipeline
- 支持 discovery-first 与手动 URI fallback

### Phase 09: MCP Prompts Workbench

**Goal**: 把 prompts 列表、参数输入和结果查看纳入 MCP 工作台
**Depends on**: Phase 08
**Plans**: 1 plan

Plans:

- [x] 09-01: Add prompts discovery, argument input, and prompt execution flow

**Details:**
- 支持 `prompts.list` 与 `prompts.get`
- prompt 参数支持结构化表单与 raw JSON fallback
- MCP 编辑区布局进一步与 HTTP 模式收敛

### Phase 10: MCP Roots Support

**Goal**: 让 roots 成为当前 MCP 调试会话的一等输入配置
**Depends on**: Phase 09
**Plans**: 1 plan

Plans:

- [x] 10-01: Add session-level roots editor and execution continuity

**Details:**
- roots 成为会话级 MCP 输入上下文
- `initialize` 声明 roots capability
- execution/history/replay 保留 roots 快照

### Phase 11: MCP Stdio Transport

**Goal**: 为单 server MCP 工作台补齐 stdio 传输与错误诊断
**Depends on**: Phase 10
**Plans**: 1 plan

Plans:

- [x] 11-01: Add stdio transport authoring, runtime execution, and diagnostics

**Details:**
- MCP 工作台支持单个本地 stdio server 配置
- HTTP / stdio 在同一 authoring 与 response pipeline 中工作
- stdio 失败提供结构化诊断信息

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Decision: v1.1 先做 `resources/prompts/roots + stdio`（Rationale: 先打透单 server MCP 工作台协议面与传输面）
- Decision: MCP 能力扩展继续沿用统一 workbench / history / replay 主链路（Rationale: 保持体验一致，避免碎片化）
- Decision: `sampling` 与多 server 管理继续延后（Rationale: 避免在 v1.1 同时扩展过多维度）

**Issues Resolved:**

- MCP 工作台从 tools-only 扩展到更完整的协议调试面
- roots 从缺失的会话上下文补齐为一等输入能力
- stdio transport 被纳入统一工作台与诊断链路

**Issues Deferred:**

- MCP `sampling`
- 多 MCP server 管理
- 更广义 AI / Agent workflow surface

**Technical Debt Incurred:**

- Phase 08–11 尚未补 `*-VALIDATION.md` Nyquist 文件，但不阻塞 milestone 归档
