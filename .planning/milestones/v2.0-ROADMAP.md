# Roadmap: ZenRequest v2.0 Local Trust & Execution Foundation

**Current milestone:** `v2.0 Local Trust & Execution Foundation`
**Status:** draft roadmap for activation after v1.3 archive
**Requirements:** `.planning/v2.0-REQUIREMENTS.md`
**Phases:** 4 planned

---

## Milestone Summary

ZenRequest 已完成并归档 v1.3 `MCP Sampling Debugging`。下一轮 milestone 不直接追求 multi-server MCP、agent workflow 或生成式主入口，而是先补齐这些能力的底层前提：execution envelope、state ownership、local persistence trust、secret-safe projection、explainable replay。

这个 milestone 的核心目标不是“多做几个新功能”，而是让 ZenRequest 在保持本地优先、轻量、可检查、可回放定位的同时，为下一轮 MCP / Agent-era 能力扩展建立稳定边界。

---

## Milestone Goals

- 定义支持 HTTP / MCP / future agent runs 的 shared execution envelope
- 明确 durable / cached / ephemeral state 的 ownership 和 restore precedence
- 提升 corruption / malformed persistence path 的可诊断性与恢复可信度
- 建立 secret-safe persistence / projection baseline
- 升级 replay/history/diagnostics 的 explainability，为后续扩展留出结构

## Sequencing Rationale

这个顺序是刻意设计的：

1. **先定义 execution model 与 state boundary**，不然后续所有改动都会落回 patchwork。
2. **再做 persistence reliability**，因为本地优先的可信度是产品根承诺。
3. **再做 secret hygiene**，确保未来 replay/export/AI context 有安全默认路径。
4. **最后升级 explainability**，因为 explainable replay 需要前面三步先把 artifact 和 persistence 边界定稳。

如果顺序打乱，最常见的失败就是：表面加了诊断或回放字段，但底层模型仍然混乱。

---

## Success Criteria

v2.0 milestone 完成时，应至少满足：

- 有 documented, implementation-ready 的 execution envelope
- durable / cached / ephemeral state ownership 明确且可测试
- corruption / malformed persistence path 不再默认 silent swallow
- secret-bearing values 有清晰的 safe persistence / projection baseline
- replay/history/diagnostics 的 explainability 明显增强，并能支撑后续 MCP / Agent-era milestone

---

## Explicit Non-Goals

本 milestone 不应扩展为：

- 完整 multi-server MCP UI / management layer
- 完整 agent workflow UX
- 通用 AI chat pane
- 团队云工作区 / 账号中心
- 插件生态 / SDK
- 大规模新协议扩展
- 面向 marketing demo 的大而全 feature bundle
